# Target a small, single-region internet service

Nano Notebook will support real remote accounts, sharing, and durable user data in one region, targeting roughly 100 registered users and 10 concurrent Agent or Source-processing jobs. It permits maintenance windows and does not promise multi-region availability or a formal 99.9% SLA. This keeps the operating model credible for a real product while reserving the project's complexity budget for Agent runtime, retrieval quality, observability, and failure recovery instead of hyperscale infrastructure.
