# Separate stable Citation identity from retrieval chunks

The Source Module will assign stable Evidence Unit addresses in each Normalized Source Artifact before retrieval indexing. Units follow source-native structural blocks and coordinates, including page regions, slide elements, document or HTML blocks, transcript intervals, and image regions; oversized blocks split only by versioned source-relative boundaries rather than model tokenization. The Retrieval Module builds deterministic, structure-aware, overlapping Retrieval Chunks that preserve structural blocks where practical, keep tables and visual evidence intact, and never cross a Source or Evidence Revision. It may replace Chunk boundaries, identifiers, vectors, or index representation as retrieval policy evolves, with concrete parameters selected through offline evaluation. Qdrant points retain the Evidence Unit coverage needed to recover authoritative content, but final Citations resolve only through the immutable Source, Evidence Revision, and one Unit or continuous Unit range, never through a Qdrant point or Retrieval Chunk identifier. This preserves historical Citation resolution across re-chunking, re-embedding, and complete index rebuilds.
