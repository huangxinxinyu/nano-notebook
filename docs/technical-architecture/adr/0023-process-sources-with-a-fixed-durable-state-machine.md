# Process Sources with a fixed durable state machine

Each uploaded or imported Source will create one top-level Source Processing Job that advances through fixed `validating`, `normalizing`, `segmenting`, `indexing`, `verifying`, and terminal stages. Every completed stage will persist its checkpoint, attempt metadata, and artifact references so a Worker can resume at the failed boundary instead of repeating upload, OCR, transcription, or embedding work. Stage writes require idempotency keys, and bounded page or media-segment parallelism may exist inside an Extractor Adapter without becoming public workflow structure. Only successful verification may publish the new Evidence Revision and Retrieval Index Version. The pipeline reuses the bounded PostgreSQL Durable Runtime and does not add arbitrary DAG nodes or a general workflow API.
