# Use REST, direct Blob Store upload, and SSE

The browser-facing Control Plane will expose JSON REST commands and queries, issue short-lived upload intents for direct presigned MinIO/S3 upload, and finalize uploaded Sources only after server-side validation of the declared object. This keeps Source bytes out of the Go request path while preserving authorization and lifecycle ownership in the Source Module. Durable Source, Job, and Agent state remains authoritative in PostgreSQL; resumable SSE projects server-to-client status and answer increments using ordered event identifiers, with reconnect reading the current durable snapshot before continuing after a cursor. Client commands such as cancellation remain REST operations. WebSockets and polling-only realtime behavior are excluded because the current product requires server-to-client streaming rather than a bidirectional session protocol. Exact Trace event semantics remain a detailed-design decision.
