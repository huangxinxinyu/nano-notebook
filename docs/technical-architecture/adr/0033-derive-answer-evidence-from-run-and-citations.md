# Derive answer evidence from Run and Citations

Assistant Messages will not store an `answer_mode`. Answer basis is established by the producing Agent Run's Grounding Outcome, pinned Run Evidence Set, verified Claim Support Records, Publication Barrier, and validated Citations. A valid Citation identity alone does not prove that its Evidence supports the adjacent claim. Composition may cite only handles returned by accepted Evidence Search Actions, deterministic checks resolve those handles and enforce scope, and a separate Model Call with an independent prompt and schema verifies support and coverage without receiving Composer rationale or retaining hidden reasoning. Composer and Verifier configurations are pinned independently, although the first release may use the same base model for both roles. Unsupported claims return to bounded research or revision and are removed or disclosed as insufficient at exhaustion. Complete non-degraded research finding zero support may select a whole-answer Model-Knowledge fallback through a fresh Source-free call, but partial support can never mix model knowledge into a Grounded Answer. Action use belongs to Checkpoints and Durable Agent Trace, while duplicated Message metadata remains removed so it cannot drift from authoritative execution and evidence state.
