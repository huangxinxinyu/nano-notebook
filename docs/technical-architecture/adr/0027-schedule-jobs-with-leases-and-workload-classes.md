# Schedule Jobs with leases and fixed Workload Classes

PostgreSQL will remain the only durable Job queue authority. Workers will claim eligible Jobs atomically with expiring leases, attempts, and heartbeats, and expired work may be reclaimed, giving at-least-once execution whose external effects and stage commits must be idempotent. Fixed interactive Agent, Source Processing, and offline Eval/Reindex Workload Classes will have separate concurrency budgets and priority so background work cannot consume all capacity; the product's one-active-Agent-Run-per-User rule is enforced when work is admitted. `LISTEN/NOTIFY` may reduce wake-up latency but polling remains the recovery path and notifications never become queue truth. Lease duration, heartbeat cadence, retries, and concurrency values remain Runtime detailed-design parameters.
