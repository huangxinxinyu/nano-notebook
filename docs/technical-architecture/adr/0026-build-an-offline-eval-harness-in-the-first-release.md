# Build an offline Eval Harness in the first release

Sprint 6 will include an offline Eval Harness that executes human-authored, versioned Eval Cases locally or in CI through the same Source, Retrieval, Models, Agent, and Citation module interfaces used by the product; no deployed production environment is required. Fixed non-sensitive fixtures and manually curated expected Evidence, equivalent Evidence sets, parsing expectations, required facts, and forbidden claims are evaluation authority. Model judges may supplement language and completeness scoring but cannot redefine ground truth or authorize promotion alone. Eval Runs identify every relevant extractor, Evidence Revision, Retrieval Index Version, model, prompt, verifier, and Agent configuration and measure parsing coverage, retrieval ranking and recall, Citation correctness and coverage, unsupported claims, grounded answer quality, latency, tokens, and cost. Promotion requires zero failures for authorization, deletion, Citation identity, and Publication Barrier invariants; every designated critical case must pass; and all other metrics must satisfy aggregate thresholds frozen before candidate comparison. Concrete thresholds remain an owning Sprint decision rather than an overall-architecture constant. Execution may use CLI and Workers; an Eval management UI, online A/B platform, automatic ground-truth generation, automatic tuning system, and separate test-only RAG implementation are excluded.
