# Normalize multimodal Sources into one text Retrieval Channel

The first release will use Gemini to extract OCR, structured descriptions, and region-linked evidence from visual regions, then index that normalized text alongside document text, HTML, captions, and audio transcripts. PDF, DOCX, and PPTX normalization first uses native text, table, image, and coordinate extraction; only content-bearing visual regions and pages without a usable text layer enter OCR or vision processing, while standalone image Sources enter as a whole. Skipped or failed regions remain explicit Evidence Coverage gaps. The release will not create a native visual-vector Collection or cross-channel fusion; original media and stable region coordinates remain available for Citations, and native multimodal retrieval can be added later without changing Source authority.
