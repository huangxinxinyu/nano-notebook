# Use hybrid retrieval, rank fusion, and reranking

The first product release will treat dense-only retrieval as an intermediate implementation milestone, not the final RAG architecture. For every query, the Retrieval Module will run dense semantic and classic BM25 lexical candidate retrieval under the same Retrieval Scope, combine their rankings without directly comparing incompatible raw scores, fetch authoritative normalized evidence, and rerank the bounded candidate set before returning it to the Agent Controller. The language-aware BM25 analyzer and parameters are versioned identically at indexing and query time; learned sparse embeddings remain deferred unless evaluation demonstrates a stable recall gap. RRF is the initial safe fusion method, while embedding and reranker models, candidate limits, fusion weights, chunking, and stopping thresholds are selected through offline evaluation. A transient reranker failure receives bounded retry and may explicitly degrade to the unchanged RRF order, with the degradation traced and no fabricated reranker scores or relaxed groundedness rules. One failed candidate channel may likewise degrade to the other only when the surviving channel completes and meets a versioned minimum-candidate policy; otherwise Retrieval fails instead of misreporting incomplete execution as insufficient Source evidence.
