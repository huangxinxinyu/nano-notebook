# Version evidence separately from retrieval indexes

An immutable Source may have multiple immutable Evidence Revisions produced by extraction, OCR, or transcription changes, and each Evidence Revision may have multiple rebuildable Retrieval Index Versions produced by chunking or embedding changes. Citations will pin `source_id`, `evidence_revision_id`, and an Evidence Unit address, while an Agent Run will pin both the Evidence Revisions in its Run Evidence Set and the corresponding Retrieval Index Version. Workers will build and validate new artifacts and Qdrant points before a PostgreSQL transaction publishes new active pointers; failed builds cannot disturb the active version, and rollback changes only those pointers. Qdrant queries will include the pinned index and evidence version in addition to the Notebook and Source filters. Old index data may be collected after active Runs release it, while old Evidence Revisions remain available as long as retained Citations depend on them; exact retention and garbage-collection policy is deferred to detailed design.
